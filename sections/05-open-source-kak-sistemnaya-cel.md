## Open Source как системная цель — подробное описание

### Зачем Prism выращивает Open Source
Open Source — это “объективная поверхность правды” для сообщества:
- позволяет закреплять знания в виде кода/шаблонов/инструментов;
- дает прозрачную историю вклада (контрибуции, ревью, релизы);
- делает сообщество полезным “вне чата” и устойчивым во времени;
- повышает качество найма и партнерств: есть доказательства компетенций.

### Что считается OSS-активом Prism
- библиотеки и пакеты (код);
- шаблоны (repo templates, CI, линтеры, security policy);
- гайды и playbooks (доки, runbooks);
- инструменты платформы (боты, TMA, автоматизация).

### Публичное vs приватное
Публичное по умолчанию, приватное — по необходимости:
- приватное: персональные данные, NDA-контент, коммерческие секреты, детали уязвимостей до фикса;
- публичное: методы, паттерны, обезличенные кейсы, инструменты.

### “No hero maintainers”
Open Source в Prism не должен держаться на одном человеке:
- минимум 2 сопровождающих на активный репозиторий;
- ротации ролей (review/triage/release shepherd);
- SLA и понятная политика “paused/stale”, если сопровождение исчезло.

### Как OSS связан с остальными процессами
- **Ячейки**: еженедельный OSS-слот, “маленькие PR”, обучение контрибуциям.
- **Гильдии**: формируют беклог OSS-улучшений, баунти, стандарты качества.
- **Governance**: ownership, лицензии, безопасность — как отдельный тип решений.
- **Токеномика**: начисления за PR/ревью/triage/релизы и баунти из казначейства.
- **Ранги**: отдельная OS-линейка для contributors/reviewers/maintainers.

### Минимальная дорожная карта OSS-слоя (MVP)
- организация репозиториев Prism (core/tools/templates);
- единые шаблоны: CONTRIBUTING, CODE_OF_CONDUCT, SECURITY, RELEASE;
- трекер задач (issues) и баунти;
- регулярный ритм: PR Clinic + Maintainers Sync.
